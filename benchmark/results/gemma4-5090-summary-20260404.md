# Gemma 4 RTX 5090 Benchmark

- Machine: `NVIDIA GeForce RTX 5090 32 GB`
- Driver: `581.42`
- Runtime: local `Transformers` BF16 path used by this project
- Prompt mode: text-only generation
- Cache: local Hugging Face cache only
- Method: one model per fresh Python process, one warmup run, then measured decode runs

## Results

| Model | Status | Official BF16 memory | Measured runs | Mean tok/s | Best tok/s | Load time | VRAM after load |
| --- | --- | ---: | ---: | ---: | ---: | ---: | ---: |
| Gemma 4 E2B | ok | 9.6 GiB | 2 x 128 target tokens | 15.96 | 16.20 | 10.69 s | 9.67 GiB reserved |
| Gemma 4 E4B | ok | 15.0 GiB | 2 x 128 target tokens | 13.23 | 13.26 | 11.10 s | 14.90 GiB reserved |
| Gemma 4 26B A4B | failed | 48.0 GiB | load only | - | - | - | failed before load |
| Gemma 4 31B | failed | 58.3 GiB | load only | - | - | - | failed before load |

## Failure details

- `Gemma 4 26B A4B`: `OSError: Le fichier de pagination est insuffisant pour terminer cette opération. (os error 1455)`
- `Gemma 4 31B`: `OSError: Le fichier de pagination est insuffisant pour terminer cette opération. (os error 1455)`

## Notes

- The generated token count was the actual decoded length, not the requested ceiling. On these runs:
  - `E2B` emitted `94` tokens per measured run.
  - `E4B` emitted `117` tokens per measured run.
- The two larger BF16 models do not fit this current local path on the machine as configured. Their failures matched the official memory table directionally.
